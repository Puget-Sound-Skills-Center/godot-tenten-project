## Effective Canvas Note

Viewport is 1920x1080 logical at scale 4.0. Effective pixel canvas: **480x270px**.
All pixel measurements below are in **logical viewport pixels** (same coordinate space as `panel.size = Vector2(...)`).
The shop panel at 260x210 occupies roughly 54% width × 78% height of the effective canvas.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, button row gaps |
| md | 12px | Margin inner padding (matches existing 12–14px margin pattern) |
| lg | 16px | Section separation |
| xl | 24px | Panel internal section breaks |
| 2xl | 32px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- VBoxContainer separation for choice buttons: 6px (matches shop vbox separation — source: player.gd line 240)
- VBoxContainer separation for dialogue elements: 8px (matches pause menu mid-range — source: pause_menu.gd line 66)
- MarginContainer inner margins: 12px left/right, 10px top/bottom (matches player.gd lines 233–236)
- NPC name/body label vertical gap: 4px (xs — tighter grouping within single speaker block)

## Typography

All sizes use `add_theme_font_size_override("font_size", N)`. No custom font file — Godot default.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| NPC Name | 14px | Bold (700) | 1.2 | Speaker label above dialogue text |
| Body | 12px | Regular (400) | 1.5 | Dialogue text content |
| Choice Button | 12px | Regular (400) | 1.0 | Choice button text labels |
| Advance Prompt | 10px | Regular (400) | 1.0 | "Press E to continue" indicator |

Source rationale:
- Pause menu title uses 16px (source: pause_menu.gd line 72). That is the largest text in the game.
- Shop feedback label uses 9px (source: pause_menu.gd line 131). That is the smallest.
- Dialogue body at 12px sits between those extremes — readable at scale 4.0 rendering.
- NPC name at 14px creates clear hierarchy without matching the "PAUSED" / panel-title weight.

Note: Godot default font does not expose named weights. "Bold" is achieved via `add_theme_font_size_override` + a bold font override if one is added later. For v1, name label uses 14px at default weight (no bold font asset exists). Visual distinction from body comes from size differential alone (14 vs 12).

## Color

All colors use Godot `Color(r, g, b, a)` notation — same as existing codebase.

| Role | Godot Color Value | Hex Equivalent | Usage |
|------|-------------------|----------------|-------|
| Dominant (60%) | `Color(0, 0, 0, 0.65)` | #000000 at 65% opacity | Full-rect overlay behind dialogue panel — matches shop overlay (source: player.gd line 218) |
| Secondary (30%) | Godot default Panel theme | ~#3d3d3d (engine grey) | Panel background — same unstyled Panel.new() used in shop and pause (source: player.gd line 222, pause_menu.gd line 49) |
| Accent (10%) | `Color.YELLOW` | #ffff00 | NPC name label only |
| Prompt dim | `Color(0.7, 0.7, 0.7, 1.0)` | #b3b3b3 | "Press E to continue" advance indicator — visually subordinate to body text |
| Destructive | not applicable | — | No destructive actions in this phase |

Accent reserved for: NPC name label only. Choice button text uses Color.WHITE (same as all other button text in project).

Color.WHITE (`Color(1, 1, 1, 1)`) applied to: dialogue body label, choice button labels, advance prompt label.
Source: pause_menu.gd line 73 — `title.add_theme_color_override("font_color", Color.WHITE)`.

Portrait placeholder ColorRect color: `Color(0.25, 0.25, 0.35, 1.0)` — dark blue-grey, distinct from panel background, signals "art goes here."

## Panel Layout Contract

### Dialogue Panel (DLG-01)

Position: Bottom strip, full viewport width.

```
[CanvasLayer layer=30]
  └─ ColorRect (overlay, full-rect, Color(0,0,0,0.65), mouse_filter=IGNORE) [visible=false]
       └─ Panel (anchors: bottom-left to bottom-right)
            size: Vector2(480, 96)
            anchor_left=0, anchor_right=1, anchor_top=1, anchor_bottom=1
            offset_top=-96, offset_bottom=0
            └─ MarginContainer (full-rect, margins: 10 all sides)
                 └─ HBoxContainer (separation=8)
                      ├─ ColorRect (portrait placeholder)
                      │    size: Vector2(72, 72)
                      │    color: Color(0.25, 0.25, 0.35, 1.0)
                      └─ VBoxContainer (size_flags_horizontal=EXPAND_FILL, separation=4)
                           ├─ Label (npc_name, 14px, Color.YELLOW)
                           ├─ Label (dialogue_text, 12px, Color.WHITE, autowrap=true)
                           └─ Label (advance_prompt, 10px, Color(0.7,0.7,0.7,1), "Press E to continue")
                                [hidden when choices are visible]
```

Panel height: 96px logical. Leaves 174px of game world visible above the panel.
Portrait: 72x72px (square). Consistent with pixel-art character scale.
Advance prompt: right-aligned within the vbox, hidden when choice buttons are shown.

### Choice Button Area (DLG-02)

When `choices` array is non-empty, advance_prompt is hidden and a VBoxContainer of Buttons replaces it:

```
VBoxContainer (choice_container, separation=6)
  ├─ Button ("Accept", size_flags_horizontal=EXPAND_FILL)
  └─ Button ("Decline", size_flags_horizontal=EXPAND_FILL)
```

Button layout: stacked vertically inside the same right-column VBoxContainer, below the dialogue text label.
Max 2 buttons. No keyboard shortcut labels (1/2) — mouse click only, matching shop button pattern.
Button text: 12px, Color.WHITE, Godot default Button theme (no custom StyleBox).

Choice input method: mouse click on Button.pressed signal — matches shop UI (source: player.gd lines 259, 271, 283).

## Interaction Contract

### Advance (no choices)

- Trigger: `Input.is_action_just_pressed("interact")` on a node with `PROCESS_MODE_ALWAYS`
- Effect: Load next node from DialogueData; if next == "" then call DialogueManager.close()
- Handled in: `_unhandled_input()` on the DialogueManager CanvasLayer

### Choose (choices present)

- Trigger: Button.pressed signal (mouse click)
- Effect: Execute choice action (set quest flag if action=="quest_offer"), then load choice.next node; if next == "" then close
- No keyboard shortcut in v1 (deferred)

### Open

- Trigger: NPC `_process()` detects `is_action_just_pressed("interact")` while player_nearby
- Effect: `DialogueManager.open(npc_id, start_node)` — sets panel visible, calls `get_tree().paused = true`
- Guard: Do not open if `player_ref.shop_open == true`

### Close

- Trigger: advance on terminal node (next == "") OR escape key (if not consumed by pause — pause is layer 50, dialogue is layer 30, pause takes priority)
- Effect: `DialogueManager.close()` — hides panel, calls `get_tree().paused = false`

### Force Close (floor advance)

- Trigger: `dungeon.gd` calls `DialogueManager.force_close()` before `reload_current_scene()`
- Effect: Same as close but no side effects (does not advance node, does not set quest flags)

## State Visibility Contract

| State | advance_prompt visible | choice_container visible | portrait visible |
|-------|----------------------|-------------------------|-----------------|
| Linear node (no choices) | true | false | true |
| Choice node (choices present) | false | true | true |
| Closed | — (panel hidden) | — | — |

Transition: instant show/hide. No fade animation in v1 (deferred — matches existing shop and pause behavior which are also instant).

## Process Mode Contract

All nodes inside the DialogueManager CanvasLayer MUST have `process_mode = Node.PROCESS_MODE_ALWAYS`.
Use the `_pa(node)` helper pattern from pause_menu.gd — replicate or import.

| Node | process_mode |
|------|-------------|
| CanvasLayer (DialogueManager itself) | PROCESS_MODE_ALWAYS |
| ColorRect overlay | PROCESS_MODE_ALWAYS |
| Panel | PROCESS_MODE_ALWAYS |
| MarginContainer | PROCESS_MODE_ALWAYS |
| HBoxContainer | PROCESS_MODE_ALWAYS |
| Portrait ColorRect | PROCESS_MODE_ALWAYS |
| VBoxContainer | PROCESS_MODE_ALWAYS |
| NPC name Label | PROCESS_MODE_ALWAYS |
| Dialogue text Label | PROCESS_MODE_ALWAYS |
| Advance prompt Label | PROCESS_MODE_ALWAYS |
| Choice VBoxContainer | PROCESS_MODE_ALWAYS |
| Each choice Button | PROCESS_MODE_ALWAYS |

Failure to set PROCESS_MODE_ALWAYS on any child = silent input failure while paused.
Source: pause_menu.gd _pa() pattern + Pitfall 1 in 02-RESEARCH.md.
