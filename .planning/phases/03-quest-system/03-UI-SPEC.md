> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

| Property | Value |

## Spacing Scale

Pixel values in logical resolution (1920×1080 before 4× scale). Multiples of 4 throughout.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Label internal padding, VBoxContainer separation (small) |
| sm | 8px | HBoxContainer separation, MarginContainer top/bottom margins |
| md | 12px | MarginContainer left/right margins (matches dialogue_manager.gd line 56) |
| lg | 16px | Quest entry height minimum, lore HUD slot height |
| xl | 24px | Quest log panel internal top padding |
| 2xl | 48px | Quest log panel minimum height per entry block |
| 3xl | 180px | Quest log panel total height (3 entries × 48px + 36px header + margins) |

Exceptions:
- HUD money label position: Vector2(8, 8) — 8px from top-left corner (existing; do not change)
- Lore artifact slot position: Vector2(8, 24) — immediately below money label at y=24
- Touch/interact targets: not applicable (keyboard/gamepad only project)

## Component Architecture

### 1. Quest Log Panel — `script/quest_log.gd`

**Node type:** `extends CanvasLayer`
**Autoload registration:** `quest_log="*res://script/quest_log.gd"` in `project.godot`
**Layer:** 29 (below dialogue layer=30, above HUD layer=10, below shop layer=20)

**Node hierarchy:**
```
quest_log (CanvasLayer, layer=29, PROCESS_MODE_ALWAYS)
└── _overlay (ColorRect)                        # full-rect dim; visibility flag
    ├── color: Color(0, 0, 0, 0.75)
    ├── anchors_preset: PRESET_FULL_RECT
    ├── mouse_filter: MOUSE_FILTER_IGNORE
    ├── visible: false (default closed)
    └── _panel (PanelContainer or ColorRect)    # right-side panel
        ├── color: Color(0.12, 0.10, 0.08, 0.95)
        ├── anchor_left: 1.0
        ├── anchor_right: 1.0
        ├── anchor_top: 0.0
        ├── anchor_bottom: 0.0
        ├── offset_left: -200                   # panel is 200px wide
        ├── offset_right: 0
        ├── offset_top: 8
        ├── offset_bottom: 228                  # 220px tall (header 28 + 3×48 + 28 hint)
        └── _margin (MarginContainer)
            ├── margin_left: 12
            ├── margin_right: 12
            ├── margin_top: 8
            ├── margin_bottom: 8
            └── _vbox (VBoxContainer, separation=4)
                ├── _header_lbl (Label)          # "Quests", 12px, Color(1.0,0.85,0.4)
                ├── _divider (ColorRect)         # Color(0.3,0.3,0.3,0.5), size=(176,1)
                ├── _entries_vbox (VBoxContainer, separation=8)
                │   ├── [entry 0] VBoxContainer  # per active quest
                │   │   ├── _name_lbl (Label)    # 12px, Color(1.0,0.85,0.4)
                │   │   └── _obj_lbl  (Label)    # 11px, Color(1,1,1,1), autowrap off
                │   ├── [entry 1] VBoxContainer
                │   └── [entry 2] VBoxContainer
                ├── _empty_lbl (Label)           # "No active quests.", 11px, grey
                └── _hint_lbl (Label)            # "[Tab] Close", 10px, grey
```

**PROCESS_MODE_ALWAYS:** Applied to `_overlay` and all descendants via `_pa()` helper (copy from dialogue_manager.gd).

**Sizing detail:**
- Panel width: 200px (offset_left = -200 from right edge)
- Panel height: 228px total
  - 8px margin_top + 16px header + 4px + 1px divider + 4px + (3 × 48px entries) + 4px + 12px hint + 8px margin_bottom = ~221px → round to 228px (multiple of 4)
- Each entry VBoxContainer: min height 48px (name label ~14px + 4px separation + obj label ~14px + 16px breathing room)

**Visibility state machine:**
- `_overlay.visible = false` → log closed, game running normally
- `_overlay.visible = true` → log open, `get_tree().paused = true`

**_refresh() logic** (called on every open):
```gdscript
func _refresh() -> void:
    # Clear existing entry children from _entries_vbox
    for c in _entries_vbox.get_children():
        c.queue_free()
    var active := []
    for qid in global.quest_state:
        var q = global.quest_state[qid]
        if q["status"] in ["active", "ready_to_complete"]:
            active.append(qid)
    _empty_lbl.visible = active.is_empty()
    for qid in active:
        _build_entry(qid)
```

**_build_entry(qid) per quest:**
- Create VBoxContainer (separation=4), add to _entries_vbox
- Name Label: `quest_data.get_quest(qid).get("display_name", qid)`, 12px, Color(1.0, 0.85, 0.4)
- Objective Label: result of `quest_manager.get_objective_string(qid)`, 11px, Color(1,1,1,1)
- If status == "ready_to_complete": append " — Return to NPC" to objective string

**Body text line height:** Objective labels (11px) are single-line, `autowrap_mode = TextServer.AUTOWRAP_OFF`. No `line_separation` override needed — Godot default line height applies. Executor note: if objective text ever wraps, add `add_theme_constant_override("line_separation", 2)` to the objective Label.

**_unhandled_input guard:**
```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if not (event is InputEventKey):
        return
    if not event.is_action_just_pressed("quest_log"):
        return
    # Guard: do not open if dialogue is active
    if dialogue_manager._panel != null and dialogue_manager._panel.visible:
        return
    get_viewport().set_input_as_handled()
    _toggle()
```

**_toggle():**
```gdscript
func _toggle() -> void:
    if _overlay.visible:
        _overlay.visible = false
        get_tree().paused = false
    else:
        _refresh()
        _overlay.visible = true
        get_tree().paused = true
```

**Input action to register in project.godot:**
```
[input]
quest_log={
"deadzone": 0.5,
"events": [Object(InputEventKey,"resource_local_to_scene":false,"resource_name":"","device":-1,"window_id":0,"alt_pressed":false,"shift_pressed":false,"ctrl_pressed":false,"meta_pressed":false,"pressed":false,"keycode":0,"physical_keycode":4194305,"key_label":0,"unicode":0,"location":0,"echo":false,"script":null)]
}
```
(physical_keycode 4194305 = KEY_TAB in Godot 4)

### 2. Lore Artifact HUD Slot — addition to `script/player.gd`

**Location in player.gd:** Added at end of `_setup_hud()` after `_hud_money_label` setup (line ~213).

**Node hierarchy:**
```
_hud_layer (CanvasLayer, layer=10)             # existing
├── _hud_money_label (Label)                   # existing, position Vector2(8,8)
└── _lore_panel (ColorRect)                    # NEW — lore artifact slot
    ├── color: Color(0.25, 0.20, 0.10, 0.9)
    ├── size: Vector2(80, 16)
    ├── position: Vector2(8, 24)               # 8px from left, 24px from top (below money label)
    ├── visible: false (default — shown only when global.items non-empty)
    └── _lore_label (Label)                    # item name
        ├── position: Vector2(2, 0)
        ├── font_size: 8px
        ├── font_color: Color(1.0, 0.85, 0.4)
        └── clip_text: true                    # prevent overflow for long names
```

**PROCESS_MODE_ALWAYS:** Not required — HUD is on player node which uses normal process mode. Panel visibility updates in `_update_hud()` which runs in `_process()` regardless.

**_update_hud() addition:**
```gdscript
var has_lore := false
for key in global.items:
    if global.items[key] > 0:
        has_lore = true
        _lore_label.text = key.replace("_", " ").capitalize()
        break
_lore_panel.visible = has_lore
```

**Visibility rule:** Panel is added to scene once in `_setup_hud()` and remains in the tree permanently. `visible` property is toggled each frame in `_update_hud()`. Do NOT add/remove from scene tree — toggle `visible` only.

**Variable declarations (add to player.gd top-level):**
```gdscript
var _lore_panel: ColorRect
var _lore_label: Label
```

### 3. Fetch Item Chest — inline in `script/dungeon.gd`

**Node type:** Area2D (runtime-built, no scene file)

**Node hierarchy:**
```
_fetch_chest (Area2D)
├── CollisionShape2D (CircleShape2D, radius=12)
├── _chest_rect (ColorRect)
│   ├── color: Color(0.55, 0.40, 0.10, 1.0)   # golden-brown
│   ├── size: Vector2(24, 24)
│   └── position: Vector2(-12, -12)            # centered on Area2D origin
└── _chest_label (Label)
    ├── text: "[E] Open"
    ├── font_size: 8px
    ├── font_color: Color(1.0, 1.0, 1.0, 1.0)
    ├── position: Vector2(-12, -24)            # above the rect
    └── visible: false (default — shown only while _player_near == true)
```

**Spawn guard (D-06):** `_spawn_fetch_chest_if_needed()` returns immediately if:
- `quest_manager.has_active_fetch_quest()` is false, OR
- `global.items.get(quest_manager.get_active_fetch_item_id(), 0) > 0` (item already held — Pitfall 3)

**Interaction:** `body_entered` signal on Area2D → `if body.has_method("player")` → track player proximity, poll `Input.is_action_just_pressed("interact")` in `_process()` (same pattern as npc.gd). On E press: `global.items[item_id] = global.items.get(item_id, 0) + 1`, then `queue_free()` the chest Area2D.

**Position:** Use `dungeon.gd`'s existing `_pick_save_position(obstacles)` or equivalent position picker. Register chest position in obstacles array after placement.

## Interaction Model

### Quest Log Toggle

| State | Trigger | Result |
|-------|---------|--------|
| Log closed, dialogue closed | Tab press | `_refresh()`, `_overlay.visible = true`, `get_tree().paused = true` |
| Log open | Tab press | `_overlay.visible = false`, `get_tree().paused = false` |
| Log closed, dialogue OPEN | Tab press | No-op (guard: `dialogue_manager._panel.visible == true`) |
| Log open, player presses E | E press | No-op (quest log does not consume E; dialogue guard already handled) |
| Scene change (floor advance) | `dungeon.gd` calls `dialogue_manager.force_close()` | Quest log must NOT be open during floor reload — executor note: call `quest_log._overlay.visible = false` and `get_tree().paused = false` in dungeon.gd before `reload_current_scene()` |

### Open/Close Animation

**Instant snap — no animation.** `_overlay.visible` toggled directly. Consistent with dialogue_manager.gd which also snaps. Rationale: pixel-art aesthetic, no tweening infrastructure in project.

### Pause State Ownership

| Panel open | Pause owner |
|------------|-------------|
| Dialogue open | dialogue_manager |
| Quest log open | quest_log |
| Both open | Impossible — quest_log._unhandled_input guards against opening while dialogue is visible |
| Pause menu open | pause_menu (layer=50, highest priority) |

**Unpause safety:** `quest_log.close()` only calls `get_tree().paused = false` when `_overlay.visible` was true and the quest log itself owns the pause. It does NOT check dialogue state on close — the open guard (Tab blocked when dialogue open) makes this safe.

### Fetch Chest Interaction

| State | Trigger | Result |
|-------|---------|--------|
| Player outside chest radius | — | Label not visible / chest inactive |
| Player enters chest Area2D | `body_entered` | Set flag `_player_near = true` |
| Player inside radius, presses E | `Input.is_action_just_pressed("interact")` in `_process()` | Add item to `global.items`, `queue_free()` chest |
| Player exits chest Area2D | `body_exited` | Set flag `_player_near = false` |
| Item already in global.items | Chest not spawned | Guard in `_spawn_fetch_chest_if_needed()` |
